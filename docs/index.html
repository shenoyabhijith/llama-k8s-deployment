⚡ KUBERNETES ORCHESTRATION FLOW

1
User: "Deploy my LLAMA server!" (applies deployment.yaml)
2
Kubernetes Control Plane: Reads the blueprint (Deployment object)
3
Control Plane → Scheduler: "I need 2 LLAMA servers. Find a place!"
4
Scheduler → GPU Node: "This GPU machine is a good fit!"
5
Control Plane → Node: "Create LLAMA Server (Pod 1 & 2)!"
6
Node: Starts the LLAMA server containers
7
Control Plane: "Are you healthy/ready?" (checks probes)
8
Pod: "Yes! I'm ready for requests!"
9
Continuous Monitoring: Kubernetes maintains 2 copies automatically
10
Auto-Recovery: If a Pod fails, Kubernetes restarts it automatically

💻 QUICK START

# Clone the repository
git clone https://github.com/shenoyabhijith/llama-k8s-deployment.git

# Navigate to the project
cd llama-k8s-deployment

# Deploy to Kubernetes
./deploy.sh

# Or use the build and push script
./scripts/build-and-push.sh

# Check deployment status
kubectl get pods -n llama-deployment

# Access the API
curl -X POST "http://localhost:8000/generate" \
     -H "Content-Type: application/json" \
     -d '{"prompt": "Tell me a story about AI"}'
